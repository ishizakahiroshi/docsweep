ファイル名を覚えていなくても 「次にやるべき作業」が先頭に出る。 AI は actions から実行可能な手だけ提案できる（不可な手は最初から選べない）。

完了しました。
plan_files-tree-search-child-filter.md を docs/local/archive/2026-06/ に移送しました（要判断 3→2、保留 2 のまま）。

MCP ツールコール

▸apply(path="…/plan_files-tree-search-child-filter.md", action="promote", to="done")
{
  "ok": true,
  "path": "docs/local/plan_files-tree-search-child-filter.md",
  "changed": {
    "state": ["watching", "done"],
    "label": ["[様子見]", "[完了]"]
  }
}
▸sweep(dry_run=false)
[
  {
    "from": "docs/local/plan_files-tree-search-child-filter.md",
    "to":   "docs/local/archive/2026-06/plan_files-tree-search-child-filter.md",
    "state": "done"
  }
]
apply は 1 ファイルに閉じた操作のみ受け付け、sweep は done/discarded のみ移送する（[様子見] や [保留] は触らない）。AI が暴走しても被害が出ない設計。

MCP ツールコール

▸summary()
{
  "projects": [
    { "name": "project-a", "todo":7, "pending":3, "fix":1 },
    { "name": "project-b", "todo":4, "pending":1, "fix":0 },
    { "name": "docsweep",  "todo":2, "pending":2, "fix":0 }
  ],
  "oldest_todo": "project-a/docs/local/plan_login-flow-refactor.md (134d)",
  "needs_fix": [
    "project-a/docs/local/bugfix_login-flow_2026-05-12.md  (H1 ラベル欠落)"
  ]
}
summary は AI のコンテキスト圧迫を避ける用。フル INDEX が要るときは index を呼ぶ。

セットアップ（参考）

# 1. docSweep を MCP extra 付きで導入
pip install 'docSweep[mcp]'

# 2. Claude Code 側の MCP 設定（~/.claude/mcp.json など）
{
  "mcpServers": {
    "docsweep": {
      "command": "python",
      "args": ["-m", "docSweep", "mcp"]
    }
  }
}

# 3. グローバル導線を一発で（セッション開始時に triage を読ませる）
docsweep inject-global --agent claude
